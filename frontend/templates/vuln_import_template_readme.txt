漏洞导入模板 - 使用说明

一、模板字段说明
================================================================================

| 列号 | 字段名     | 是否必填 | 说明                                           |
|------|------------|----------|------------------------------------------------|
| A    | 漏洞标题   | 是       | 漏洞的简短描述，长度5-200字符                   |
| B    | 漏洞描述   | 否       | 漏洞的详细技术描述                             |
| C    | 漏洞类型   | 否       | 见下方"漏洞类型字典值"                         |
| D    | 风险等级   | 是       | 见下方"风险等级字典值"                         |
| E    | 来源类型   | 是       | 见下方"来源类型字典值"                         |
| F    | 资产编码   | 是       | 必须在系统中存在，且与资产编码完全一致          |

================================================================================

二、字典值参考
================================================================================

【风险等级】(risk_level)
  critical - 严重
  high     - 高危
  medium   - 中危
  low      - 低危

【漏洞类型】(vuln_type)
  injection    - 注入类
  xss          - XSS跨站
  csrf         - CSRF跨站请求伪造
  file-upload  - 文件上传漏洞
  info-leak    - 信息泄露
  auth-bypass  - 权限绕过
  ssrf         - SSRF服务端请求伪造
  rce          - 远程代码执行
  other        - 其他

【来源类型】(vuln_source)
  manual      - 人工发现
  scanner     - 扫描器发现
  pentest     - 渗透测试发现
  code-review - 代码审计发现
  threat-intel- 威胁情报
  other       - 其他

================================================================================

三、注意事项
================================================================================

1. 必填字段（A、D、E、F列）不能为空
2. 资产编码（F列）必须与系统中已有资产的编码完全匹配
3. 字典值字段（C、D、E列）必须使用上述表格中的代码值，不是显示名称
4. 日期格式：YYYY-MM-DD，如 2025-01-15
5. 文件编码：UTF-8（如果导入后中文乱码，请检查文件编码）
6. 建议先导入少量数据测试，确认格式正确后再大量导入

================================================================================

四、导入结果说明
================================================================================

导入完成后，系统会返回：
- totalCount: 总记录数
- successCount: 成功导入的记录数
- failCount: 导入失败的记录数
- errors: 失败记录的详细错误信息列表

如果有任何一条记录导入失败，都会显示在错误列表中，您可以根据
错误信息修改后重新导入。

================================================================================
